Lane Detection Car

Description

Status: Finished


In late 2023, I began working on a lane detection car project that uses computer vision and a technique called pixel summation to determine the curvature of the road or path. The system works by drawing a line in the middle of the video feed and analyzing which side has more pixels. If the left side has more pixels, the curve is to the left. If the right side has more pixels, the curve is to the right. If both sides have an equal number of pixels, the path is straight. However, this approach has a drawback, if the car is not perfectly centered on the road, it might incorrectly detect a curve due to an uneven pixel distribution. To address this, I implemented a histogram to find the midpoint of the road or path.


After completing the computer vision system, I moved on to the physical build of the car. The car is built on a basic a crylic base and uses a single H-bridge to control the motors for movement and direction changes. It is controlled by a Raspberry Pi, when the Raspberry Pi signals the car to move forward, it sends a signal to the H-bridge to turn all the motors clockwise. For backward movement, the H-bridge turns the motors counterclockwise. To turn, the car deactivates one side's motors while the other side remains active.


Unfortunately, I was unable to make the car fully reliant on the lane detection software because the Raspberry Pi camera malfunctioned due to compatibility issues with the newly released Debian software. Since my project depended on the camera for image analysis, I couldn’t fully integrate the physical system with the computer vision system. If the camera had worked, the project would likely have succeeded, as the computer vision system accurately detected curves, and all that was needed was to relay the curve location to the physical system to adjust the H-bridge accordingly.
